# Endispute — Dynamic Websites Pipeline

Three production-grade dynamic websites for Endispute.com.au, all sharing one
content source of truth, each with its own visual personality and animation
language.

## The pipeline

| Layer | Tool | Why |
|---|---|---|
| Framework | **Next.js 15** (App Router) | static export friendly, SSR-ready, best DX |
| Language | **TypeScript** | type-safe content schema reused across all 3 sites |
| Styling | **Tailwind CSS v4** | utility-first, inline arbitrary values, JIT |
| Animations | **Motion (Framer Motion)** `motion/react` | 120fps spring physics, gestures, scroll, layout |
| Components | **shadcn/ui** + **21st.dev community patterns** | hand-tuned, copy-owned, accessible primitives |
| Icons | **lucide-react** | clean, consistent, tree-shakeable |
| Runtime | **Bun** | 10× faster install + dev server |

## Why this stack vs. the static HTML approach

The 5 static HTML versions in `../Endispute_site/versions/` are great for fast
review and easy email-to-client. **They cannot do**:

- scroll-linked parallax, sticky-then-snap sections, or shared layout animations
- gesture-driven UIs (drag, pinch, hover-trail)
- live data (a real form, a typewriter, a CMS-fed news feed)
- per-route code splitting and image optimisation
- routing, SEO meta per page, sitemap generation

These are the things `framer-motion`, `next/image`, `next/link`, and
shadcn's headless primitives unlock. Hence this second pipeline.

Each site is an **independent Next.js app** with its own `package.json` —
clone any one of them out and ship it.

## Run any site locally

```bash
cd site-1-aurora        # or site-2-courtroom / site-3-mosaic
bun install
bun dev                 # http://localhost:3000
```

## Build for production / static export

```bash
cd site-1-aurora
bun run build           # next build (with output: 'export')
# Static HTML+JS is now in ./out/ — drop into any static host
```

## Update content for ALL three sites at once

Edit `shared/content.ts`. Each site imports from there via a relative path,
so a single edit propagates everywhere.

## Stack research notes

### Framer Motion / Motion for React
- Imported as `motion/react` (v12+, formerly `framer-motion`)
- Key APIs we use: `<motion.div animate={}>`, `whileHover`, `whileInView`,
  `useScroll`, `useTransform`, `AnimatePresence`, `layout`, `layoutId`
- Hardware-accelerated where possible (Web Animations API), JS fallback for
  spring physics + interruptible keyframes

### 21st.dev
- Public registry of curated React+Tailwind+shadcn components (free to
  copy-paste, you own the code)
- Their "Magic" MCP server is a paid AI component generator — we **don't**
  need it, we're hand-building informed by their patterns
- Patterns we lift: animated bento grids, marquee/ticker, hero gradient
  meshes, magnetic buttons, scroll-snap testimonials, command-K palette

### "UI UX Pro Max" — what we encode by hand
- Type scale: 4 sizes max per page, geometric ratio
- Colour: 1 primary + 1 accent + neutrals (no rainbow)
- Spacing: 4/8/16/32/64 px ladder
- Motion: spring stiffness 100–300, damping 20–40, never linear easing
- Accessibility: `prefers-reduced-motion` respected on every animation,
  focus-visible rings, AA contrast minimum
